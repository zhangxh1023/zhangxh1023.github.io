看了一下源码, 找了一下原因.

就是由于 java 解析嵌套的 x-www-form-urlencoded 类型的参数的规则, 和我们 stringify 默认方法的规则, 不一样.

Node.js 一般都是用的 qs.stringify 来序列化参数. qs.stringify 有两个参数, 一个是需要序列化的 js 对象, 一个是选项. 其中可以加入 { allowDots: true } 参数, 来代替默认的选项, 就可以让 Spring Boot 正确的解析. 所以我们应该直接传入一个序列化好了的字符串, 然后发送给后端, 而不是让 urllib, axios, superagent 之类的请求库帮我们处理参数.
