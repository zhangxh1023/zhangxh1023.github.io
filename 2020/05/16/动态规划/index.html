动态规划

发表于 2020-05-16

最近无聊, 又开始尝试刷起了 leetcode, 然而, 菜的抠脚, 解题几乎全靠题解…
特别是 动态规划 的问题, 题目难度只要是 medium, 就基本可以直接看题解了…

所以, 大概整理记录几道题的思路. 希望下周刷题或者是周赛, 能有一丝丝的进步嘿嘿嘿…

最早开始了解动态规划, 是从知乎的一个问题, 什么是动态规划 中了解到的. 就像这个回答中所说的一样, 如何判断一个问题, 能用 动态规划 去解决呢?

未来与过去无关, 能将大问题拆成几个小问题, 且大问题的最优解可以由小问题的最优解推出

斐波那契数列

网上搜到的讲动态规划的资料, 很多都是以 斐波那契数列 开始的. 搜了一下 leetcode, 正好也有这道题.

其实这道题的示例里面就已经写的很清楚了.

示例来源: https://leetcode-cn.com/problems/fibonacci-number

假如用 F(n) 表示第 n 个 斐波那契数列.
根据 斐波那契数列 的定义, 也可以推出:

F(n) = F(n - 1) + F(n - 2)

即, 当 n > 2 时, F(n) 的值, 只和它的前两位数有关. 这就是它的 无后效性, 未来与过去无关.

而我们可以很容易的就得到:

F(0) = 0, F(1) = 1

所以, 我们可以用一个数组, 保存它的 dp table. 然后一步步推出第 n 个 斐波那契数列.

三角形最小路径和

这是第120题.
给一个类似于三角形的数组:

从上而下移动, 每次只能移动到相邻节点. 求经过路径的数字最小和.
其实这些题目, 遇到的时候, 我都是看了标签, 带有 动态规划 的标签, 我才知道, 哦~~ 这题可以用 动态规划 来解决😯.

希望以后能够熟能生巧, 看到题目就知道思路~~.

假设数组为 N. 数组第一行的最小和为 F(N[0]), 走到数组第一行的第一个, 最小值为 f(N[0][0]). 那么 F(N[N.length - 1]) = min{f(N[N.length - 1][0]), ... , f(N[N.length - 1][N[N.length - 1].length - 1])}. 也就是, 走到最后一行的最小路径和, 为走到最后一行, 每一个坐标都是当前坐标的最小路径和. 这些最小路径和中的最小值, 就是整个数组的最小路径和.

而如何求得走到最后一行的每一个坐标的最小路径和呢?

比如我们要得到到达最后一行数组, [4, 1, 8, 3] 的最小路径和, 去除边界情况 4, 3. 到达 1 的最小路径和, 就等于到达前一个数组 [6, 5, 7] 中的 6, 5 的最小路径和 +1.
同理, 到达 8 的最小路径和, 就等于到达 5, 7 的最小路径和 +8.
而比如边界情况 4, 就只等于到达 6 的最小路径和加上它本身 4 了.

也就是说, 要求得到达某一个点的最小路径和, 就等于能移动到该点的点的最小路径和加上它本身.

那么, 已经达到了前面所说的 无后效性. 并且, 把大问题, 拆分成小问题, 先通过 N[0], 求得到达 N[1] 的最小路径和, 再求得到达 N[2], N[3], .... 的最小路径和, 直到到达 N[N.length - 1]. 便能得到 F(N[N.length - 1]), 也就是整个数组的最小路径和.

至于该题中的说明:

如果你可以只使用 O(n) 的额外空间（n 为三角形的总行数）来解决这个问题，那么你的算法会很加分。

因为我偷懒, 用一个 triangle.length * triangle[0].length 的数组的额外空间, 来生成 dp table, 保存 动态规划 计算过程中生成的每一个坐标点的最小路径和. 其实里面有很多数据不需要都保存, 可以直接用新的值覆盖, 节省空间.这个就后面有缘再优化了吧….
