2018.7面试小结

发表于 2018-06-27 | 更新于 2018-06-28

　　准备跑路走人了,然后也开始了悲惨滴面试之路.俗话说得好,面试造导弹,工作拧螺丝~….哎嗨,然而我就喜欢那些造导弹的玩意儿.因为好玩儿(哈哈哈哈..

　接下来,就记录一些有(我)意(不)思(会)的面试题吧~…..

　1. 浏览器 304 的缓存机制是怎么样的, post　请求的 304 ,服务器端会接收到请求的数据吗?

　　说实话,不会! 对于浏览器的 304 缓存机制,我的理解就是,客户端发送get请求给服务端,服务端判断缓存还能用,返回 304 ,浏览器就会读取缓存.当然了,我也是这么回答的.于是面试结束的路上,随手网上搜了一波….(捂脸….hhhhh
　　说到状态码 304 的缓存,也应该了解一下 200 OK (from cache) 它返回的状态码是200,但是,这次请求,浏览器是没有和服务器确认,直接使用了浏览器的缓存,所以返回的是 from cache .
　　而 304 它的产生过程呢?
　　当浏览器访问一个文件时,浏览器会先判断,这个文件在缓存是否存在,是否过期,如果可用,那么是不会发送请求,而是直接用浏览器之前缓存下来的文件.
　　如果浏览器判断,缓存文件不可用,将会发送请求给服务器.而服务器将会判断,服务器上的文件是否已经改动,如果发现没有改动并且浏览器的缓存还能使用,那么将会返回 304 ,让浏览器继续使用缓存文件.

　　而浏览器是如何判断缓存是否还可用的呢?那就涉及到了 http 头信息的设置,这也是另一道题, http 有哪些头信息….
