惯例一段废话…
由于一些不可描述的政策, 一大堆的爬虫相关的公司被请去了喝茶. 由于我们公司非常依赖这些第三方的数据( 剩下的数据是由我们组进行抓取, 提供给自己公司内部使用 ), 所以业务受到了影响…然后裁员bulabulabula….最后我们组就剩我一个人负责爬虫业务喽….刺激. 所以既然只剩我一个人, 感受到了前所未有的忙碌, 紧张, 以及… 自由

当然了, 其实之前是有类似的一个项目的, 只不过长年累月的更新迭代, 之前的人也都懒得维护, 就东一个接口, 西一个接了…

1.连接数据库(mongodb)

由于之前的代码习惯, 成功连接数据库, MQ 等, 都喜欢输出一句 log (connect to xxxxxxx), 并且捕捉错误信息.
而文档中推荐的异步 provider, 代码如下: 

import * as mongoose from 'mongoose';

export const databaseProviders = [
  {
    provide: 'DATABASE_CONNECTION',
    useFactory: (): Promise<typeof mongoose> =>
      mongoose.connect('mongodb://localhost/nest'),
  },
];

它通过了 useFactory 来提供了一个异步的方法. 然后在创建模型的地方通过 DATABASE_CONNECTION 这个 token 来注入 useFactory 返回的连接. 由于我项目中会用到多种, 并且多个数据库. 所以我不应该用 mongoose.connect 来连接 mongo (因为这样只默认把连接保存在 mongoose.connection 上, 有多个连接的时候会覆盖), 而是应该用 mongoose.createConnection 来创建一个连接, 并且保存这个连接.

但是…我突然发现, mongoose.createConnection 的返回值类型, 不是一个 Promise, 而是 mongoose.Connection. 难道mongoose.createConnection 这个方法, 不是异步的吗?
通过简单的写了几行代码尝试: 

const mongoose = require('mongoose')

const connection = mongoose.createConnection('mongodb://localhost:27017/test')
connection.on('open', () => {
  console.log('connection open')
})
console.log('-----------------flag-----------------')

结果自然是异步的…

经过搜索发现, 虽然 mongoose.connect 返回的是一个 Promise<typeof mongoose>, 但是 mongoose.createConnection 返回的, 并不是我们平时使用的那个 Promise.

就像前面所说的, mongoose.connect , 会自动把连接保存到 mongoose.connection 上, 而 mongoose.createConnection 是直接返回一个数据库连接.
但尽管返回的是一个 mongoose.Connection, 但是却还是能把他当做一个 Promise 来使用. 

mongoose.createConnection('mongodb://localhost:27017/test').then(connection => {
  // do somethings...
  connection.model('Test', schema);
})

猜测应该是 mongoose 内部实现了一个 Promise, 实现不用等待连接成功就能先初始化 model : 

const connection = mongoose.createConnection('mongodb://localhost:27017/test')
connection.model('Test', schema);

而由于 mongoose.connect 是把连接赋值给 mongoose.connection 的, 所以能直接取到, 并且进行初始化 model 的任务, 如果 mongoose.createConnection 返回的和 mongoose.connect 一样是一个 Promise, 那么将需要等 Promise.resolve() 之后, 才能拿到 mongoose.connection 的值, 这样就会比较浪费时间.

所以我觉得, 这也不是个问题, 就算 await 的对象没有 .then() 方法, 它也不会报错.

以上参考链接: https://thecodebarbarian.com/whats-new-in-mongoose-5-improved-connections

其实感觉这一部分, 和 Nest.js 好像并没有多大关系hhh..但是我还是把它写进来了.就当作是一个记录吧🙄
